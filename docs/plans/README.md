# Implementation plans

Plans preserve task state for work that spans more than a tiny change. They connect requirements, code, tests, risks, and completion evidence.

- Start from `PLAN_TEMPLATE.md`.
- Keep one active owner for each vertical slice.
- Update status and evidence as work proceeds.
- Link approved architecture changes to an ADR.
- Archive completed plans in place; their history is useful engineering evidence.

The current milestone plan is `release-0.1-transactional-core.md`.
